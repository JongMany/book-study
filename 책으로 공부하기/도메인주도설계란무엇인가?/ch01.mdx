# 도메인 주도 설계란 무엇인가?

**소프트웨어**

- 현실 세계의 프로세스를 자동화하거나 비즈니스 문제를 해결하기 위해 개발됨
- **`자동화된 비즈니스 프로세스`나 `현실 세계의 문제`**가 소프트웨어의 도메인임

좋은 소프트웨어를 만들기 위해서는

- 그 소프트 웨어가 무엇에 관련된 것인지 알아야 함

소프트웨어의 전반적인 목적

- 특정 도메인의 일이 더 잘 굴러가도록
  ⇒ 이는 소프트웨어가 대상 도메인과 조화를 이뤄야 가능

하나의 도메인 = 세상의 어떤 것

도메인 모델

- 다이어그램 그 자체가 아닌 다이어그램이 전달하고자 하는 아이디어
- 지식에서 선택적으로 추상화하여 엄격하게 조직화한것
- 정보를 조직화, 체계화하고 작은 부분으로 나누고, 그 조각들을 다시 논리적 모듈로 그룹화한 다음, 한 번에 하나씩 선택해서 다뤄야 함

모델

- 대상 도메인에 대한 내부적 표현
  - 설계와 프로세스 전반에 걸쳐 반드시 필요

모델 관련 지식을 공유하는 방법

1. 시각적 도구 활용
   1. 다이어그램, 유스케이스, 그림, 사진
2. 글
   1. 도메인의 비전을 기술
3. 언어
   1. 도메인의 특정 이슈를 서로 알리고 나눌 언어를 만들기

⇒ **`모델로 의사소통할 필요가 있음`**

### 도메인 지식 쌓기

예시) 비행 항로 제어 시스템 구축 프로젝트

```jsx
상황:
	- 특정 시점에 수천대의 비행기들이 지구 전체에 걸쳐 공중에 떠있음
	- 비행기들은 자기 경로를 따라 목적지를 향해 날고 있고,
		공중에서 충돌하지 않게 하는 것이 중요
프로젝트:
	- 특정 지역 위의 모든 비행을 추적하고, 예정 경로를 따르고 있는지, 충돌 가능성이 있는지 여부를 판단하는 감시 시스템
```

도메인

- 항공 교통 모니터링

도메인 전문가

- 항공 교통 관제사
- 관제사들과 이야기 함으로써, 필수 정보들을 골라내 일반화 ⇒ 필요한 모델을 구축
