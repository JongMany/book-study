# 도우미 추출

루틴 속 코드 중에서 목적이 분명하고 나머지 코드와는 상호작용이 적은 코드를 만난 경우, 그 코드 블록을 추려내고 '도우미'로 추출한 후 이름을 붙임  
➡️ 리팩터링 기법 중 '메서드 추출' 리팩터링임

코드를 정리할 때 자동화 도구를 사용하자

1. 큰 루틴 안에서 몇 줄을 변경해야 하는 경우, 해당하는 줄들을 도우미로 추출하고 도우미 안의 내용만 변경한 다음 적절하다가 판단한 뒤에 도우미를 호출하는 문장에 반영하기

AS-IS

```
routine()
  ...그대로 두는 코드
  ...바꾸는 코드
  ...그대로 두는 코드
```

TO-BE

```
routine()
  ...그대로 두는 코드
  helper()
  ...그대로 두는 코드

helper()
  ...그대로 두는 코드
```

이 작업은 응집도 높은 요소를 만들기 위한 방식이다.

2. 시간적 결합을 표현하는 경우(a 함수가 b 함수보다 앞서 호춣되어야 하는 경우), 도우미 추출을 사용하면 코드를 이해하기 쉬워짐

AS-IS

```
foo.a()
foo.b()
```

TO-BE

```
ab()
  a()
  b()
```

#### 도우미를 만들다보면 재사용성을 고려하게 되며 인터페이스는 문제를 생각하도록 이끄는 도구가 된다.

#### 우리가 만든 설계 요소에 새로운 이름을 붙일 수 있을 때, 새로운 인터페이스가 만들어지는 것임
