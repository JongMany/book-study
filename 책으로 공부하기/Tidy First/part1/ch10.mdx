# 명시적인 매개변수

변경하려는 코드를 읽는 중에 루틴에서 다루고 있는 일부 데이터가 명시적으로 전달하지 않는 것을 발견할 경우, 입력을 명확하게 하려면?

#### 👍 루틴을 나누자! 앞부분에서 매개변수 값을 채운 후, 뒷부분에서 명시적으로 전달하기

#### 매개변수를 명시적으로 드러나게 만든 다음 함수를 연쇄적으로 호출할 수 있게 준비하면 코드를 이해하기 쉬워지고 테스트하기도 쉬워짐

예시 상황
map에서 매개변수가 블록으로 전달되는 경우가 많음.  
이런 경우 코드를 읽으면서 어떤 데이터가 필요한지 알기 어렵기 때문에 매개변수를 변경하여 암묵적으로 사용하는 등의 남용이 발생할 수 있음.

AS-IS

```
params = { a: 1, b: 2}
foo(params)

function foo(params)
  # params.a, params.b를 사용하는 코드 // 여기서 어떤 값을 사용하는지 파악해야만 함
```

TO-BE

```
function foo(parmams)
  foo_body(params.a, params.b)

function foo_body(a, b)
  # a, b를 사용하는 코드
```

foo를 나누면 명시적 매개변수 정리를 적용할 수 있음

이 외에도 여러 경우가 존재함

- 중괄호가 여러 개로 둘러쌓인 코드에서 환경변수를 사용하는 경우.
