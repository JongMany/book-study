# 설명하는 변수

코드의 어떤 표현식은 성장함. 처음에는 작게 시작했더라도 시간이 흐르면 코드는 커지게 됨

#### 👍 어렵게 크고 복잡한 표현식을 이해했다면, 전체에서 일부 표현식을 추출한 후 표현식의 의도가 드러나도록 변수 이름을 만들어 할당하기

AS-IS

```
return new Point(
  ...긴 표현식,
  ...다른 긴 표현식
);
```

TO-BE

```
int x = ...긴 표현식
int y = ...다른 긴 표현식
return new Point(x, y);
```

설명하는 변수를 사용하여 코드를 이해하기 쉽게 만들기

- 변수명을 통해서 가독성을 높일 수 있음.

✅ 코드 정리에 대한 커밋과 동작 변경에 대한 커밋을 분리하자!
