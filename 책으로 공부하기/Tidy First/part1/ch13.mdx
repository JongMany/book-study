# 하나의 더미

코드가 여러개의 작은 조각으로 나뉘어져 있을 때 가독성이 떨어질 수 있음.  
필요한 만큼의 코드를 하나의 더미처럼 느껴질 때까지 흩어진 코드를 모아야 함

#### 코드를 만들 때 가장 큰 비용이 들어가는 일은 코드 작성이 아니라 읽고 이해하는데 드는 비용임

코드 정리를 선행하면 더 작은 조각 단위로 결합을 제거하는 길을 제시하여 응집도를 높일 수 있고, 한 번에 머릿속에 기억하고 있어야 할 코드의 상세 내용을 줄여줌

작은 코드 조각을 지향하는 목적은 코드를 한 번에 조금씩 이해할 수 있도록 하는 것이지만 때때로, 코드를 더 작은 조각으로 나누는 것이 오히려 코드를 이해하기 어렵게 만들 수 있음

#### 명확성을 위해서는 코드를 한데 모아 이해하기 어려운 부분은 추출하여 새롭게 정리하는 것이 좋음

#### 다음과 같은 증상이 발생한다면 하나의 더미로 묶어야 함

- 길고 반복되는 인자 목록
- 반복되는 코드 (반복 조건문)
- 도우미에 대한 부적절한 이름
- 공유되어 변경에 노출된 데이터 구조

더 작은 조각을 지향한다고 말하면서, 동시에 하나의 더미로 만드는 일은 앞뒤가 맞지 않는 느낌이 들기도 하지만 직접해보면 만족을 느낄 수 있음
