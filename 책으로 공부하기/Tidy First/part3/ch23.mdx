# 구조와 동작

소프트웨어는 2가지 방식으로 가치를 만듦

1. 현재 소프트웨어가 하는 일 ➡️ 시스템 동작 (급여 계산, 주문 배송, 알림 발송 따위의 시스템 동작)
2. 미래에 새로운 일을 시킬 수 있는 가능성

동작을 규정하는 방식은 2가지임

1. 입출력 쌍
2. 불변 조건

동작은 가치를 만든다.

시스템의 구조는 동작에 영향을 미치지 않음.  
하나의 큰 기능으로 소프트웨어를 구성하는 것과 작은 기능을 여러 개로 만들어 소프트웨어를 구성하나 동작의 결과는 같음.  
그러나 구조는 미래의 기회를 만든다. 구조에 따라 급여 계산에 새로운 국가를 추가하는 것이 쉬워질 수도 있고, 어려워질 수도 있음.

하지만 구조는 동작처럼 뚜렷하게 드러나지 않기 때문에, 사람들은 동작 변경과 달리 구조 변경에 대해 어려워 함.  
구조 변경과 동작 변경 모두 가치를 만들어내지만 근복적으로 다름. (한마디로 되돌릴 수 있는 능력, 가역성 부분에서 다름)

### ChatGPT의 정리

구조 변경과 동작 변경의 가역성 차이는 다음과 같은 몇 가지 측면에서 달라질 수 있습니다.

1. 변경의 범위와 영향
   동작 변경: 주로 특정 기능이나 프로세스의 변경을 의미합니다. 코드 수정, 알고리즘 개선, 인터페이스 변화 등이 여기에 포함됩니다. 동작 변경은 일반적으로 특정 영역에 국한되어 있어, 변경 사항을 되돌리거나 수정하는 것이 비교적 쉽습니다.
   구조 변경: 시스템의 아키텍처나 데이터 모델, 큰 틀의 설계를 변경하는 것을 의미합니다. 구조 변경은 시스템 전체에 영향을 미치기 때문에 되돌리기가 더 어렵고, 영향을 받는 영역도 더 넓습니다.
2. 변경의 복잡도
   동작 변경: 특정 기능을 수정하거나 추가하는 것은 비교적 단순하고, 변경 사항을 테스트하고 롤백하는 것도 상대적으로 간단합니다. 따라서 동작 변경의 가역성은 높다고 할 수 있습니다.
   구조 변경: 구조를 변경하려면 시스템의 다양한 부분을 재설계하고, 전체 시스템의 일관성을 유지해야 합니다. 구조 변경 후 발생할 수 있는 문제를 예측하고 해결하는 데 더 많은 노력이 필요하며, 따라서 구조 변경의 가역성은 낮습니다.
3. 의존성 및 결합도
   동작 변경: 특정 모듈이나 컴포넌트의 동작만을 변경하는 경우, 해당 모듈에만 주로 영향을 미치며, 다른 부분과의 의존성이나 결합도가 낮습니다. 이에 따라 변경 사항을 되돌리는 것이 비교적 쉽습니다.
   구조 변경: 시스템의 전체적인 구조를 변경할 때는 다양한 모듈 간의 의존성과 결합도가 높아질 수 있습니다. 이로 인해 변경 사항을 되돌리는 것이 더 복잡하고 어려워집니다.
4. 데이터와 상태
   동작 변경: 데이터 모델이나 저장된 상태에 큰 변화를 주지 않으므로, 변경 사항을 되돌리는 것이 상대적으로 간단합니다. 데이터 일관성을 유지하기 쉽습니다.
   구조 변경: 데이터 모델 자체를 변경하거나 데이터 저장 방식을 바꾸는 경우, 데이터 마이그레이션이 필요할 수 있으며, 변경 사항을 되돌리기 위해서는 데이터 복구와 같은 추가적인 작업이 필요합니다.
   결론적으로, 동작 변경은 주로 특정 기능이나 프로세스의 변경을 포함하며, 이러한 변경은 국한된 영역에 영향을 미치므로 되돌리기 비교적 쉽다고 할 수 있습니다. 반면에, 구조 변경은 시스템 전체에 영향을 미치고, 더 많은 의존성과 결합도를 수반하기 때문에 되돌리기가 더 어렵고 복잡합니다. 이러한 이유로 구조 변경의 가역성이 더 낮다고 할 수 있습니다.
