# 대칭으로 맞추기

코드는 마치 유기체처럼 성장함. 필요한 모든 코드를 한꺼번에 작성할 수는 없기 때문임

코드가 유기체로 성장하면 같은 문제라도 다른 방식으로 해결되는 경우가 많음.  
동일한 문제를 푼 코드가 서로 다른 형태로 작성된다면 일관성을 위해 코드를 통일하는 작업이 필요함

초기화 지연(Lazy Initialization)

- 프로그램 실행 과정에서 변수 초기화를 위해 사전에 별도 준비가 필요하거나 하는 것과 같은 이유로 변수 선언 즉시 초기화하지 않게 하는 최적화 기법을 말함

다양한 방식으로 구현된 초기화 지연 코드

```
// 1
foo()
  return foo if foo not nil
  foo := ...
  return foo

// 2
foo()
  if foo is nil
    foo := ...
  return foo

// 3. 기교를 가한 코드
foo()
  return foo not nil
    ? foo
    : foo := ...

// 4. 할당문이 수식인 경우(까다로움)
foo()
  return foo := foo not nil ? foo : ...

// 5. 조건이 잘 드러나지 않는 수준의 코드
foo()
  return foo := foo || ...
```

여러 패턴이 있지만 한가지 방식으로 정하는 것이 좋음.  
다른 방식으로 작성된 코드는 하나로 통일함

때로는 공통성이 있는데도 세부사항에 묻혀 드러나지 않는 경우, 비슷해보이지만 같지 않는 루틴을 찾은 후에 같은 부분들 속에 다른 부분을 분리해나가는 작업을 진행해야 함
