# 응집도를 높이는 배치

코드를 읽다가 변경해야 할 동작을 찾았더니 여러 곳에 흩어져 있는 코드를 수정해야 한다면, 이는 코드의 응집도가 낮다는 신호.  
코드의 순서를 바꿔서 변경할 요소들을 가까이 두면 된다.

두 루틴에 결합도가 있다면 두 루틴을 가까이 배치하라.  
두 파일이 결합도가 있다면 두 파일을 하나의 디렉토리에 두자

결합도를 제거할 수 있다면 제거하자. 그러나 비용 상 응집도를 높이는 게 더 효율적임

- 결합도 제거(Decoupling) 비용 + 변경 비용 < 겳합도(Coupling)에 따른 비용 + 변경 비용

- 결합도 제거 비용을 높이는 요소
  - 어떻게 결합도를 제거해야할지 모름
  - 결합도를 제거하는 방법을 알아도 변경 비용이 높음
